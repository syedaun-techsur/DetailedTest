**`src/app.js`** — Express application setup:
```javascript
require('dotenv').config();
const express = require('express');
const helmet = require('helmet');
const methodOverride = require('method-override');
const path = require('path');
const taskRoutes = require('./routes/tasks');

const app = express();

// Security headers (TechArch §5.4)
app.use(helmet({ contentSecurityPolicy: false })); // CSP off for v1 inline scripts

// View engine: EJS with auto-escaping
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'views'));

// Body parsing
app.use(express.urlencoded({ extended: false }));

// Method override: support _method in POST body (TechArch §2.1)
app.use(methodOverride('_method'));

// Root redirect to /tasks
app.get('/', (req, res) => res.redirect('/tasks'));

// Task routes
app.use('/', taskRoutes);

// 404 handler
app.use((req, res) => {
  res.status(404).render('errors/404', { message: 'Page not found.' });
});

// 500 error handler — never leak stack traces (TechArch §5.5)
app.use((err, req, res, next) => {
  console.error(err.stack);
  const status = err.status || 500;
  if (status === 503) {
    return res.status(503).render('errors/503', { message: 'The application is temporarily unavailable. Please try again later.' });
  }
  res.status(status).render('errors/500', { message: 'An unexpected error occurred. Please try again.' });
});

module.exports = app;
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→app | HTTP requests from browser entering Express routes and middleware |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information disclosure | `src/app.js` error handler (line: `app.use((err, req, res, next) => {...})`) | mitigate | Error handler renders only the user-facing message string from the error catalog — never `err.stack` or `err.message` — to the browser. Stack traces are logged server-side via `console.error(err.stack)` only. EJS templates receive a developer-controlled `message` string, not raw error objects. |
| T-01-02 | Elevation of privilege | `src/app.js` `helmet` middleware | mitigate | `helmet` is registered as the first middleware in `src/app.js`, setting `X-Content-Type-Options: nosniff`, `X-Frame-Options: SAMEORIGIN`, `X-XSS-Protection: 1; mode=block`, `Referrer-Policy: same-origin` on every response. CSP is intentionally disabled for v1 inline delete-confirmation scripts (accepted residual risk owned by application team). |
| T-01-03 | Tampering | `src/routes/tasks.js` method-override via `_method` POST body field | accept | `method-override` reads `_method` from the POST body (not query string) to override to PUT/DELETE. The application has no auth and intentionally permits any browser to perform any CRUD operation (TechArch §5.1). Misuse risk is mitigated by the confirmation dialog on DELETE (F03). Residual risk owned by operations team via firewall/VPN access control. |
</threat_model>
